# Data Inventory

## Status

Documentation-only inventory. This file does not implement retention, deletion, anonymization, TTL, schema changes, Firestore Rules changes, Cloud Functions, or client behavior.

## Scope

This inventory maps the current user-scoped Firestore data paths used by Quantum Finance. It is intended to support future retention, deletion, privacy, and access-review work without changing runtime behavior.

Do not add real personal data, secrets, service account material, CPF, real import hashes, financial descriptions, real amounts, or before/after payloads to this document.

## Cross-Cutting Notes

- This inventory describes current documentation assumptions and observed paths only.
- Firestore subcollections are independent documents; deleting a parent document does not automatically delete nested subcollections.
- Any future total account deletion must be implemented as a controlled server-side workflow using Admin SDK privileges, with dry-run, export/backup, sanitized logs, tests, rollback planning, and LGPD validation.
- User-facing deletion does not always mean physical deletion. For financial records, soft delete and retained history preserve integrity and auditability.
- Operational documentation must use placeholders only, such as `<uid>`, `<transactionId>`, `<amount_cents>`, and `<ticket-id>`.
